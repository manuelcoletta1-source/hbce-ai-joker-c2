# 1. Introduzione

Le infrastrutture digitali contemporanee stanno diventando sempre più autonome.

Sistemi di intelligenza artificiale, piattaforme digitali distribuite, robotica e ambienti cyber-fisici stanno progressivamente assumendo un ruolo operativo all'interno di settori critici della società moderna.

Tra questi:

- energia
- sanità
- trasporti
- finanza
- infrastrutture tecnologiche
- sistemi spaziali

Questa trasformazione introduce una nuova sfida fondamentale.

L'attribuzione delle azioni digitali.

Nei sistemi tecnologici complessi spesso non è possibile determinare con certezza quale entità abbia generato una determinata azione.

Questo problema diventa critico quando i sistemi digitali assumono un ruolo operativo.

---

# 2. Il problema dell'attribuzione digitale

La maggior parte delle infrastrutture digitali contemporanee utilizza modelli di autenticazione basati su:

- account
- token
- sistemi di accesso

Questi modelli sono adeguati per l'accesso ai servizi digitali, ma non sono progettati per garantire l'attribuzione operativa delle azioni.

In ambienti nei quali operano:

- sistemi AI
- robot autonomi
- piattaforme automatizzate
- infrastrutture digitali distribuite

diventa necessario un modello diverso.

Un modello nel quale ogni azione digitale possa essere associata a una identità operativa persistente.

---

# 3. Identity Primary Record (IPR)

HBCE introduce il concetto di **Identity Primary Record (IPR)**.

L'IPR rappresenta un identificatore persistente associato a una entità operativa.

Le entità possono includere:

- esseri umani
- agenti di intelligenza artificiale
- robot
- droni
- infrastrutture digitali
- nodi tecnologici

Esempio di identità AI:

IPR-AI-0001

Attraverso l'IPR le azioni digitali possono essere associate a entità identificabili.

---

# 4. Pipeline di evidenza deterministica

Il modello HBCE utilizza una pipeline operativa progettata per generare evidenza verificabile.

request → identity → evidence → verification

### request

Una richiesta operativa viene generata da un utente o da un sistema.

### identity

La richiesta viene associata ad una identità IPR.

### evidence

Durante l'esecuzione il sistema produce metadati operativi.

### verification

Le evidenze generate possono essere verificate attraverso registri o sistemi di audit.

Questo approccio consente la ricostruzione delle operazioni digitali.
