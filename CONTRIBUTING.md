## Introduzione

Questo repository contiene il codice e la documentazione dell'applicazione **AI JOKER-C2**, parte dell'ecosistema tecnologico HBCE.

Il progetto è attualmente sviluppato come ambiente di ricerca dedicato allo studio di:

- identità operative persistenti
- sistemi AI coordinativi
- infrastrutture digitali verificabili
- modelli di governance tecnologica

I contributi sono benvenuti se allineati con gli obiettivi del progetto.

---

# 1. Principi di sviluppo

Lo sviluppo del progetto segue alcuni principi fondamentali.

- chiarezza architetturale
- tracciabilità delle modifiche
- separazione tra interfaccia e logica applicativa
- documentazione tecnica coerente

Ogni modifica dovrebbe migliorare la comprensibilità del sistema.

---

# 2. Struttura del repository

La struttura del repository segue il modello Next.js.

api/ chat.js

globals.css vercel.json

Documentazione:

README.md ARCHITECTURE.md PROTOCOL.md SECURITY.md

---

# 3. Standard dei commit

I commit devono utilizzare messaggi chiari e descrittivi.

Formato consigliato:

type(scope): description

Esempi:

feat(api): add Joker-C2 chat endpoint refactor(ui): rebuild interface layout docs(repo): add architecture documentation chore(repo): update gitignore

---

# 4. Processo di contributo

Per contribuire al progetto:

1. Fork del repository
2. Creazione di un branch dedicato
3. Implementazione della modifica
4. Commit con messaggi chiari
5. Apertura di una Pull Request

Le Pull Request dovrebbero includere una breve descrizione del cambiamento proposto.

---

# 5. Linee guida per il codice

Il codice dovrebbe seguire alcune pratiche base:

- evitare duplicazione di logica
- mantenere funzioni modulari
- commentare le parti architetturalmente rilevanti
- mantenere separata la logica API dall'interfaccia

---

# 6. Documentazione

Ogni modifica architetturale significativa dovrebbe essere accompagnata da aggiornamenti nei documenti:

ARCHITECTURE.md PROTOCOL.md

Questo aiuta a mantenere il progetto comprensibile nel tempo.
