## 7. Information Classes

AI JOKER-C2 should classify information before circulation or exposure.

Suggested information classes:

```txt
PUBLIC_INFORMATION = public-safe content or metadata
INTERNAL_INFORMATION = internal project or runtime information
CONFIDENTIAL_INFORMATION = restricted business, institutional or legal material
SENSITIVE_INFORMATION = protected operational or personal information
SECRET_INFORMATION = keys, tokens, credentials or private security material
PERSONAL_INFORMATION = personal data or identifiers
CIVIC_INFORMATION = civic participation, eligibility or public decision data
DEMOCRATIC_CHOICE_INFORMATION = vote or civic-choice content
SECURITY_INFORMATION = security findings, logs, vulnerabilities or infrastructure details
CRITICAL_OPERATIONAL_INFORMATION = live system or critical infrastructure information
UNKNOWN_INFORMATION = insufficiently classified information
```

Unknown information must not be treated as public-safe in sensitive contexts.

---

## 8. Information Handling Rules

AI JOKER-C2 should follow these handling rules.

```txt
PUBLIC_INFORMATION = may be exposed publicly
INTERNAL_INFORMATION = may be used inside controlled runtime context
CONFIDENTIAL_INFORMATION = minimize, review and restrict
SENSITIVE_INFORMATION = minimize, redact or reference
SECRET_INFORMATION = do not expose, do not repeat, recommend rotation if leaked
PERSONAL_INFORMATION = minimize, purpose-limit and avoid unnecessary reproduction
CIVIC_INFORMATION = minimize and preserve democratic safeguards
DEMOCRATIC_CHOICE_INFORMATION = do not link to identity; escalate or block depending on context
SECURITY_INFORMATION = defensive-only handling and review
CRITICAL_OPERATIONAL_INFORMATION = escalate before operational reliance
UNKNOWN_INFORMATION = classify before exposure or treat conservatively
```

The general rule is:

```txt
Classify before circulation.
Minimize before storage.
Trace before proof.
Review before reliance.
Expose only what is public-safe.
```

## 10. Controlled Circulation

Information circulation must be governed.

Controlled circulation means:

```txt
the source is identified;
the purpose is known;
the context is classified;
the data class is known;
the receiving layer is appropriate;
the exposure boundary is defined;
the event is traceable;
the proof receipt exists when needed;
the exchange is reviewable.
```

Uncontrolled circulation includes:

```txt
permissionless data exchange;
unbounded document sharing;
public exposure of internal metadata;
private payload circulation without purpose;
secret propagation;
democratic choice exposure;
security-sensitive leakage.
```

MetaExchange is the primary module for controlled circulation.

MetaExchange must remain governed, access-controlled and policy-aware.
