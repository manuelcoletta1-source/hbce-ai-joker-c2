# Panoramica

Il sistema Joker-C2 non è progettato come un singolo software monolitico.

È concepito come una rete di componenti modulari denominati:

UFO — Unità Funzionale Opponibile

Ogni modulo UFO rappresenta un micro-sistema capace di collegare:

- infrastrutture fisiche
- sistemi digitali
- contesti operativi e legali

Lo scopo dei moduli UFO è rendere i processi tecnologici:

- misurabili
- verificabili
- auditabili
- certificabili nel tempo

# Motore di stabilità Λ

Tutti i moduli UFO condividono una metrica comune.

Λ rappresenta lo stato di stabilità del sistema.

Condizione ideale:

Λ ≈ 1

Quando Λ devia dalla finestra operativa si genera una instabilità.

Il sistema Joker-C2 può quindi:

- rilevare anomalie
- prevedere deviazioni
- applicare correzioni operative
